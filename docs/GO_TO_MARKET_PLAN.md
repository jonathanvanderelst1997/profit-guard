# Go-to-market plan

## Positioning

Margin Sentinel is a Shopify catalog margin leak scanner.

It helps merchants answer:

- Which variants are losing money?
- Which variants are below our minimum gross margin?
- Which products still have missing cost data?
- What should we fix first?

Do not position it as full net-profit analytics until order-level costs, shipping, fees, refunds, taxes, and ad spend are included.

Do not use the old Profit Guard name publicly. A different app with that exact name is already live in the Shopify App Store, and it owns an order-profit story. Margin Sentinel should own the catalog/SKU margin story.

## Ideal customer profile

Best early customers:
- Shopify merchants with 500+ physical SKUs.
- Stores with frequent supplier cost changes.
- Teams that import CSV price lists or use bulk product edits.
- Agencies that manage catalog operations for several Shopify merchants.
- Wholesale/B2B or multi-vendor catalogs where one bad SKU can hide for weeks.

Weak fits:
- Tiny stores with fewer than 50 products.
- Digital-only stores without meaningful unit cost.
- Stores already deeply embedded in a full profit analytics suite unless they need a simpler catalog guardrail.

## Acquisition channels

1. Shopify App Store search
   - Keywords: margin, product margin, COGS, cost of goods, low margin, profit leak, supplier costs, SKU cost.
   - Screenshots should show: Action Center, CSV import preview, suggested minimum price, weekly alerts.
   - First screenshot should say "Know what to fix first", not "Dashboard".

2. Direct merchant outreach
   - Target stores with large catalogs and visible physical products.
   - Ask about supplier cost updates and bulk edit mistakes.
   - Offer a beta margin leak scan.

3. Shopify agencies
   - Pitch as a client catalog QA tool before sales, promos, or supplier updates.
   - Agency offer: free Growth for first client, then $39/store.

4. Content
   - "How to find Shopify products selling below cost."
   - "Why Shopify profit reports need clean product costs."
   - "Supplier price list changed? Run this Shopify margin check."

## First 30 days

Week 1:
- Finalize App Store listing, privacy policy, screenshots, and reviewer demo script.
- Recruit 10 beta merchants or agencies.

Week 2:
- Watch 5 merchants run their first scan.
- Record where they hesitate: costs, CSV, target margin, export, alerts.
- Add the top UX fix.

Week 3:
- Publish 2 short help articles and 1 product demo video.
- Send 50 targeted outreach emails.

Week 4:
- Submit to Shopify App Store or continue beta if review assets are not ready.
- Track: installs, first scan rate, cost import rate, issues found, export rate, alert enabled rate.

## Product metrics

- Install to first scan conversion.
- First scan to finding found conversion.
- Cost import rate.
- Export rate.
- Weekly alerts enabled rate.
- Trial to paid conversion.
- Number of issues fixed, if merchants report it manually.

## App Store screenshot sequence

1. Action Center: "Fix this product first."
2. Findings table: issue, margin, gap, suggested min price.
3. Supplier CSV import preview.
4. Weekly alerts setup.
5. Pricing/simple plans.

## Beta offer

"Free Growth plan for 60 days in exchange for one 20-minute feedback call and permission to use anonymized learnings."
